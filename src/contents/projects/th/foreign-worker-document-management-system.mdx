## 🎯 วัตถุประสงค์

โปรเจกต์นี้เป็นส่วนหนึ่งของรายวิชา **01418321-65 การวิเคราะห์และการออกแบบระบบ** ภาควิชาวิทยาการคอมพิวเตอร์ มหาวิทยาลัยเกษตรศาสตร์ โดยพัฒนาระบบขึ้นมาเพื่อแก้ปัญหาให้กับ บริษัท เอ็มทีเอฟ แมนเพาเวอร์ จำกัด ธุรกิจผู้จัดการเอกสารนำเข้าแรงงานต่างด้าว (พม่า, ลาว, เวียดนาม) สู่ประเทศไทยอย่างถูกกฎหมาย ซึ่งเป็นธุรกิจของบุคคลใกล้ชิดกับสมาชิกในกลุ่ม

เนื่องจากการจัดการเอกสารนำเข้าแรงงานมีหลายขั้นตอนและต้องติดตามสถานะอย่างต่อเนื่อง ซึ่งต้องการความถูกต้องและแม่นยำสูง กระบวนการทำงานรูปแบบเดิมจึงก่อให้เกิดปัญหา ดังนี้:

- **การจัดเก็บข้อมูลไม่เป็นระบบ:** ข้อมูลนายจ้างและแรงงานปะปนกัน ทำให้การค้นหาและเข้าถึงข้อมูลใช้เวลานาน และเสี่ยงต่อการสูญหายของข้อมูล

- **ข้อจำกัดด้านอุปกรณ์:** ข้อมูลถูกจัดเก็บในคอมพิวเตอร์เพียงเครื่องเดียว ไม่สามารถเข้าถึงหรือตรวจสอบข้อมูลจากอุปกรณ์อื่นได้

- **ขาดระบบติดตามสถานะ:** ทำให้ตรวจสอบได้ยากว่าเอกสารของแรงงานคนใดใกล้หมดอายุ หรือดำเนินการถึงขั้นตอนใดแล้ว

- **ความซ้ำซ้อนในการทำงาน:** ต้องกรอกข้อมูลซ้ำหลายครั้งในกระบวนการต่างๆ เนื่องจากไม่มีบันทึกและดึงข้อมูลอัตโนมัติ

- **ข้อผิดพลาดในการออกเอกสารทางการเงิน:** การออกบิลหรือใบเสร็จต้องตรวจสอบด้วยตนเอง และไม่มีการจัดเก็บข้อมูลค่าใช้จ่ายอย่างเป็นระบบ

เพื่อแก้ปัญหาดังกล่าว ผมและสมาชิกในกลุ่มจึงได้ริเริ่มพัฒนา **Foreign Worker Document Management System (FWDMS)** หรือ "ระบบจัดการเอกสารแรงงานต่างด้าว" โดยมีเป้าหมายหลักในการนำเทคโนโลยีเข้ามาเพิ่มประสิทธิภาพการทำงานให้เป็นระบบ รวดเร็ว และแม่นยำมากยิ่งขึ้น

## 🧑🏻‍💻 บทบาทและความรับผิดชอบ

ในโปรเจกต์นี้ ผมรับหน้าที่เป็น Project Leader โดยมีความรับผิดชอบหลักดังนี้:

- **Project Management:** บริหารจัดการโปรเจกต์ แจกจ่ายและติดตามความคืบหน้าของงานภายในทีม รวมถึงเป็นผู้ประสานงานหลักกับตัวแทนของธุรกิจ

- **Design & System Analysis:** ออกแบบประสบการณ์และส่วนติดต่อผู้ใช้งาน (UX/UI) และออกแบบโครงสร้างฐานข้อมูล (ER-Diagram)

- **Development:** พัฒนาส่วนหน้าบ้าน (Frontend) ทั้งหมด และพัฒนาระบบยืนยันตัวตนและกำหนดสิทธิ์ (Authentication & Authorization) ในส่วนของ Backend

## 🛠️ กระบวนการศึกษาและพัฒนาระบบ

ขอบเขตของรายวิชานี้มุ่งเน้นที่การทำความเข้าใจบริบทของธุรกิจ วิเคราะห์ปัญหา และออกแบบวิธีแก้ปัญหาด้วยเทคโนโลยีอย่างเป็นระบบ โดยทีมงานได้ผ่านกระบวนการตั้งแต่:

- การเก็บรวบรวมความต้องการ (Requirement Gathering) จากตัวแทนธุรกิจโดยตรง

- การวิเคราะห์และออกแบบระบบผ่านแผนภาพ เช่น AS-IS & TO-BE Business Process, Sequence Diagram, Class Diagram และ ER-Diagram ก่อนลงมือพัฒนาซอฟต์แวร์จริง

> **🙏** โปรเจกต์นี้คงไม่อาจสำเร็จลุล่วงไปได้ด้วยดี หากขาดความอนุเคราะห์จากตัวแทนพนักงานของบริษัท ที่ได้สละเวลาอันมีค่าในการให้ข้อมูลเชิงลึกและให้ความร่วมมือกับทีมของเราอย่างเต็มที่ ทางคณะผู้จัดทำขอขอบพระคุณมา ณ ที่นี้

## 💻 ฟีเจอร์หลักของระบบ

ระบบได้แบ่งสิทธิ์การเข้าถึงออกเป็น 2 บทบาทหลัก ดังนี้:

### 1. พนักงานผู้ดำเนินการ (นายหน้า)

- **จัดการกระบวนการทำงาน:** รองรับเวิร์กโฟลว์หลักของธุรกิจ เช่น การขึ้นทะเบียนแรงงานใหม่ และการต่ออายุใบอนุญาต

- **จัดการข้อมูลนายจ้าง:** บันทึกและบริหารจัดการข้อมูลของนายจ้าง

- **จัดการข้อมูลแรงงาน:** บันทึกและบริหารจัดการข้อมูลของแรงงานหรือแรงงานต่างด้าว

- **จัดการเอกสารทางการเงิน:** จัดการระบบการออกใบเสร็จรับเงิน

- **ระบบติดตามสถานะ:** ติดตามสถานะการดำเนินงานและวันหมดอายุเอกสารของแรงงานแต่ละราย

### 2. ผู้ดูแลระบบ

- **จัดการบัญชีพนักงานผู้ดำเนินการ:** บริหารจัดการสิทธิ์และข้อมูลของพนักงานผู้ดำเนินการในระบบ

## 🔁 กระบวนการทำงานของทีม

เพื่อให้การพัฒนาระบบเป็นไปอย่างราบรื่นและตรงตามความต้องการของผู้ใช้งานจริง ในสัปดาห์ที่มีความคืบหน้าสำคัญ ผมจะนัดหมายตัวแทนของบริษัททุกวันจันทร์ เวลา 13:00 น. เพื่อนำเสนอผลงานที่ได้พัฒนาในสัปดาห์นั้น ๆ พร้อมรับฟังข้อเสนอแนะ เพื่อนำมาปรับปรุงระบบให้ตอบโจทย์การใช้งานให้ได้มากที่สุด

นอกจากนี้ ภายในทีมจะมีการประชุมอัปเดตงานกันทุกวันจันทร์ในช่วงเวลา 18:00 น. หรือ 19:00 น. เพื่อรายงานความคืบหน้าของแต่ละคน วางแผน แจกจ่ายงานในระยะถัดไป รวมถึงร่วมกันแก้ไขปัญหา ที่แต่ละคนพบเจอในแต่ละวัน เพื่อให้โปรเจกต์เดินหน้าไปได้อย่างต่อเนื่อง

## 💡 สิ่งที่ได้เรียนรู้

- **การเข้าใจธุรกิจต้องมาก่อนการลงมือพัฒนา:** ในช่วงแรก ทีมมุ่งเน้นที่ความรวดเร็วในการพัฒนาระบบโดยที่ยังไม่เข้าใจ Business Logic อย่างถ่องแท้ ผลลัพธ์คือระบบที่ได้ไม่สามารถแก้ปัญหาได้ตรงจุด เราจึงต้องยอมถอยกลับมามองภาพรวมใหม่ ศึกษาและทำความเข้าใจกระบวนการธุรกิจอย่างละเอียดอีกครั้ง โดยได้รับความช่วยเหลือจากตัวแทนบริษัทที่ช่วยอธิบาย Workflow ให้ ซึ่งทำให้ตระหนักว่าความเข้าใจในธุรกิจคือรากฐานที่สำคัญที่สุดของการพัฒนา

- **การสื่อสารเชิงรุกคือหัวใจสำคัญ:** ช่วงแรกของการพัฒนา ผมใช้วิธีแจกจ่ายงานและรอตรวจทานความคืบหน้าเมื่อถึงกำหนด ซึ่งทำให้ไม่ทราบปัญหาที่เกิดขึ้นระหว่างทาง ภายหลังผมจึงได้ปรับวิธีการบริหารจัดการ โดยเน้นการสื่อสารระหว่างวันให้มากขึ้น เช่น การทักไปสอบถามสมาชิกในทีมว่า _"งานที่ได้รับมอบหมายเป็นอย่างไรบ้าง?"_, _"ติดปัญหาตรงไหนไหม?"_ หรือ _"ต้องการความช่วยเหลือหรือไม่?"_ การปรับเปลี่ยนวิธีนี้ช่วยลดคอขวดของการทำงานและทำให้โปรเจกต์เดินหน้าได้อย่างมีประสิทธิภาพมากขึ้น

## 👾 วิธีติดตั้งและรันโปรแกรม

### Backend Setup

1. โคลน Repository ของ Backend

```sh
git clone https://github.com/aockgonnabefree/mtf-backend
```

2. เข้าไปยังโฟลเดอร์โปรเจกต์

```sh
cd mtf-backend
```

3. สร้างไฟล์ Environment Variables โดยคัดลอกไฟล์ต้นแบบจาก `.env.example`

> อย่าลืมเข้าไปตรวจสอบและแก้ไขค่าตัวแปรในไฟล์ .env ให้ตรงกับสภาพแวดล้อมของคุณ

```sh
cp .env.example .env
```

4. เริ่มต้นการทำงานของ Database ผ่าน Docker

```sh
docker compose up -d db
```

5. เปิด IntelliJ เพื่อรันโค้ด Java Spring Boot

### Frontend Setup

1. โคลน Repository ของ Frontend

```sh
git clone https://github.com/thg1rb/mtf-frontend
```

2. เข้าไปยังโฟลเดอร์โปรเจกต์

```sh
cd mtf-frontend
```

3. ติดตั้ง Dependencies (แพ็กเกจที่จำเป็น)

```sh
bun install
```

4. รัน Development Server

```sh
bun run dev
```
